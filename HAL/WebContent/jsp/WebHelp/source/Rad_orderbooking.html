Order Booking

Screen Overview
This screen captures the order book for investigation for the out patient. This screen will be used for maintaining order booking for investigations. This screen will contain information of Clinical notes, Prescribed by and test name.



Service No:
User can write Service no based on service no Patient name list will appear. It can't remain blank.
Patient Name:
User can select patient name from the list, it can't remain blank.
Visit No:
Based on selection Patient Name latest visit id will display, it can't remain blank.
Order No:
It is system generated number, it can't remain blank.
Order date:
It is system generated date, it can't remain blank.
Order Time:
It is system generated time, it can't remain blank.
Service No:
It is patient service no, it can't remain blank.
HIN No:
It is patient HIN No, it can't remain blank.
Visit No:
It is patient visit no, it can't remain blank.
Patient Name:
It is patient name, it can't remain blank.
Sex:
It is patient sex, it can't remain blank.
Age:
It is patient age, it can't remain blank.
Martial Status:
It is patient martial status, it can't remain blank.
Religion:
It is patient religion, it can't remain blank.
Test Type:
User can select test type, it contains regular and urgent, Default value is regular, and it can't remain blank.
Clinical Notes:
User can enter patient clinical notes, it can't remain blank.
Prescribed by:
User can select from the list, it can't remain blank.
Test Name:
User can write order test details, it can't remain blank.
Template:
User can create his own Department wise templates.
Provisional Diagnosis:
It will fetch from OPD/Ward.
Submit (Button):
By clicking on submit button user can save the data.
Reset (Button):
Clicking on reset button, will clear the current or existing data.
