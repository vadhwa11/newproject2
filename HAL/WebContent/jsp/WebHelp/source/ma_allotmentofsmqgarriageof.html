Allotment of SMQs / Garages Officers

Screen Overview
This screen will capture for Allotments of SMQ for officers.



Pool Code:
User can select the pool code from the given list, this will come from pool code screen, it will display only officer’s pools and rank wise .It can't remain blank. Based on selection of pool code all vacant SMQ’s list and pending register officer’s list will display.
Allotment order file No:
It is system generation number, user can’t change number.
Allotment order file Date:
It is system generated date, user can’t change date and it is present date.
GRID (Display):
Based on search criteria data will display in the form of table. All pending registration details will display with Registration No, Registration Date, Service No, Service person name, And rank.
SMQ No:
User can select the SMQ No from the given list, this will come from SMQ screen, and it will display all vacated SMQ’s list. It can't remain blank.
Type of Allotment:
User can select from list, it contains 1) TMQ / Non Status, 2) Status, 3) Medical Grounds 4) Admin Grounds and 5) Retention
Type of garage:
User can select the garage No from the given list, this will come from garage screen, it can remain blank.
Garage No:
User can select the garage No from the given list, this will come from garage screen, it can remain blank.
Allotment Date:
It is system generated date, user can’t change date and it is present date.
HO/TO Date:
It is system generated date, user can’t change date and it is present date.
Remarks:
User can write remarks and observations.
Save (Button):
By clicking on save button user can save the data.
Reset (Button):
Clicking on reset button, will clear the current or existing data.
All.Order (Button):
Clicking on Allotment order button, user can print allotment order details.
