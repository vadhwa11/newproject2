Work Completion Details

Screen Overview
This screen captures the Work Completion details against complaint Number and it captures completion date, docket number and Administrator Remarks.



Complaint Code or Description:
On the basis of the search requirement, user can either choose code or complaint description.
Department:
User can select the department from the given list, this will come from department screen, and it can't remain blank.
Complaint type:
User can select the complaint type from the given list, this will come from complaint type screen, and it can't remain blank.
Textbox and Search (Button):
Textbox is to write the value you are searching or looking for. Once it's done you can click on search button to get the result of your search.
GRID (Display):
Based on search criteria data will display in the form of table. At a time only five (5) records will be displayed. To navigate or look for the next record pagination (the system of numbering pages) will be there.
OK Button and Text Box:
If user wants to switch directly to any particular page no, then he can simply enter the page no. in the given text box and click on Go Button.
Complaint Register:
When user clicks grid the Complaint Number, Complaint Date, Complaint Time, Complaint type, Complaint Criteria, Service No, Service Person name, Department, Building no, Complaint Details will display. All are read only user can’t change the existing value.
Completion Date:
It is system generation date, user can’t change date, it can't remain blank.
Completion Time:
User can write allocate person name. it can't remain blank.
Allocated To:
User can write allocate person name. it can't remain blank.
Total Cost:
User can write cost of particular work. it can't remain blank.
Complaint Attended Date:
User can select attended date, it should be greater than or equal to Completion Date, it can't remain blank.
Complaint Attended Time:
User can select attended time, it can't remain blank.
Docket No:
User can write docket number, it can't remain blank.
Additional Remarks:
User can enter any observation, it can't remain blank.
Save (Button):
By clicking on save button user can save the data.
Reset (Button):
Clicking on reset button, will clear the current or existing data.
Back (Button):
Clicking on Back button, user will move to the previous screen.
