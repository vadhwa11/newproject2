Order Booking IPD Search

Screen Overview
This screen captures the order book for investigation for the Admitted patient. This screen will be used for maintaining order booking for investigations. This screen will contain information of Clinical notes, Prescribed by and test name.






Search (Button):
User can search the records based on Service No, A&D No, HIN No and Ward Wise.
GRID (Display):
Based on search criteria data will display in the form of table. At a time only five (5) records will be displayed. To navigate or look for the next record pagination (the system of numbering pages) will be there.
OK Button and Text Box:
If user wants to switch directly to any particular page no, then he can simply enter the page no. in the given text box and click on Go Button.
Order No:
It is system generated number, it can't remain blank.
Order date:
It is system generated date, it can't remain blank.
Order Time:
It is system generated time, it can't remain blank.
Admission No:
It is patient admission number, it can't remain blank.
Service No:
It is patient service no, it can't remain blank.
Relation:
It is patient Relation, it can't remain blank.
Service Person Name:
It is Service person Name, it can't remain blank.
Patient Name:
It is patient name, it can't remain blank.
Sex:
It is patient sex, it can't remain blank.
Age:
It is patient age, it can't remain blank.
Ward:
It is patient present admitted ward, it can't remain blank.
Test Type:
User can select test type, it contains regular and urgent, Default value is regular, and it can't remain blank.
Clinical Notes:
User can write patient clinical notes, it can't remain blank.
Prescribed by:
User can select from the list, it can't remain blank.
Test Name:
User can order test details, it can't remain blank.
Template:
User can create his own Department wise templates.
Provisional Diagnosis:
It will fetch from OPD/Ward.
Submit (Button):
By clicking on submit button user can save the data.
Reset (Button):
Clicking on reset button, will clear the current or existing data.
