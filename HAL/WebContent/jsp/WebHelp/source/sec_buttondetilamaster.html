Button Details Master

Screen Overview
This screen captures the Button Details. Where ever button level security required user can create button level security.



Button Name or Form Name:
On the basis of the search requirement, user can either choose Button Name or Form Name.
Textbox and Search (Button):
Textbox is to write the value you are searching or looking for. Once it's done you can click on search button to get the result of your search.
GRID (Display):
Based on search criteria data will display in the form of table. At a time only five record will be displayed. To navigate or look for the next record a pagination (the system of numbering pages ) will be there.
Go Button and Text Box :
If user wants to switch directly to any particular page no., the he can simply enter the page no. in the given text box and click on Go Button.
Button Name:
User can write button name, it can't remain blank.
Form Name:
User can write form name, it can't remain blank.
Css Class Name:
User can write CSS class name, it can't remain blank.
Formula Used:
User can write formula used, it can remain blank.
URL:
User can write URL name of form, it can't remain blank.
Add (Button):
By clicking on Add button user can save the data.
Update (Button):
By clicking on Update button user can modify the existing data.
Inactive (Button):
By clicking on inactive button user can inactive the existing data.
Reset (Button):
Clicking on reset button, will clear the current or existing data.
Back (Button):
Clicking on Back button, will move user to the previous screen.
