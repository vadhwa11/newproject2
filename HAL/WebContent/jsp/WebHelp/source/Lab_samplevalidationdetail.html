Sample Validation Detail

Screen Overview
It capture details related to sample validation like patients details along with service personnel details, the name of employee who has collected the sample and who is validating the sample. And also the list of samples accepted and validated along with date and time of sample collection.



Sample Validation Date:
It is system date and today’s date, user can’t change date and it can't remain blank.
Sample Validation Time:
It is system time and present time, user can’t change time and it can't remain blank.
Investigation Accepted By:
Default login person name will display, if user wants to change name user can change. It can't remain blank.
Diag No:
User can enter serial number manually.
Copy (Button):
Same Diagnosis number copies for all investigations.
Rejected Reason:
User can enter rejected reason.
Copy (Button):
Same reason copies for all investigations.
Diag No:
User can enter diagnosis number, it can't remain blank.
Test Code:
Based on order number test code will display.
Test Name:
Based on order number test name will display.
Sample:
Based on test sample will display from diagnosis master.
Quantity:
Based on order sample quantity will display from diagnosis master.
Accepted:
User can accept single test or multiple tests.
Rejected:
User can reject single test or multiple tests if test is rejected then reason should be written.
Additional Remarks:
User can write any observations.
Submit (Button):
By clicking on save button user can save the data.
Reset (Button):
Clicking on reset button, will clear the current or existing data.
Back (Button):
Clicking on Back button, user will move to the previous screen.
