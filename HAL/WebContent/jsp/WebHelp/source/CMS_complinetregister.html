Complaints Register

Screen Overview
This screen captures the details of Complaint register details of Complaint type, Complaint criteria, Complaint person name, and department and Complaint details.



Complaint Code or Complaint Name:
On the basis of the search requirement, user can either choose complaint code or complaint name.
Textbox and Search (Button):
Textbox is to write the value you are searching or looking for. Once it's done you can click on search button to get the result of your search.
GRID (Display):
Based on search criteria data will display in the form of table. At a time only five (5) records will be displayed. To navigate or look for the next record pagination (the system of numbering pages) will be there.
OK Button and Text Box:
If user wants to switch directly to any particular page no, then he can simply enter the page no. in the given text box and click on Go Button
Complaint Number:
It is system generation number, it can't remain blank.
Complaint Date:
It is system generation date, user can’t change date, it can't remain blank.
Complaint Time:
It is system generation time, user can’t change time, it can't remain blank.
Maximum Limit:
User can set maximum limit of each work category, it can't remain blank.
Complaint Type:
User can select the complaint type from the given list, this will come from Complaint screen, and it can't remain blank.
Complaint criteria:
User can select complaint criteria it contains new or repeat. It can't remain blank.If it is Repeat user can write previous 3 complaint details.
Complaint Location:
User can select location whether it is department or residential. It can't remain blank. User can select multiple departments, it can't remain blank. If it is department: Department should be selectable, it can't remain blank. If it is residential: SMQ No should be selectable, it can't remain blank.
Complaint Details:
User can enter complaint details, it can't remain blank.
Save (Button):
By clicking on save button user can save the data.
Reset (Button):
Clicking on reset button, will clear the current or existing data.
Back (Button):
Clicking on Back button, user will move to the previous screen.
