Pending Sample Validation

Screen Overview
This screen captures the Pending Sample Validation details.






Patient Search:
From Date:
User can write From date details, it can't remain blank.
To Date:
User can write to date details, it can't remain blank.
P Type:
User can select P Type from the given list.
Service No.:
User can write Service number details.
HIN No.:
User can write HIN Number details.
A&D No.:
User can write A & D number details.
Order By:
User can select order by from the given list.
Patient First Name:
User can write Patient First Name details.
Patient Last Name:
User can write Patient last Name details.
Search (Button):
User can search the records based on the From Date,To Date,P Type,Service No.,HIN No.,A&D No.,Order By,Patient First Name,Patient Last Name.
GRID (Display):
Based on search criteria data will display in the form of table. At a time only five (5) records will be displayed. To navigate or look for the next record pagination (the system of numbering pages) will be there.
OK Button and Text Box:
If user wants to switch directly to any particular page no, then he can simply enter the page no. in the given text box and click on OK Button.
Sample Validation Page: 
This page will show the details of the patient selected from the grid like - order date, order time, order no., order by, service type, service no, service status,relation,ser. per. name,rank,unit,trade, Hin no. patient name, sex, age, marital status, sample validation date, sample validation time etc.
Validated By Drop Down Box:
User can select validated by from the drop down box, this is mendatory field.
Accepted and Rejected Option Buttons:
User can select the Accepted and Rejected from the option buttons.
Remarks Text Box:
User can write his remarks into this text box.
Submit Button:
By clicking on the submit button user can save the record.
Reset Button:
Clicking on reset button, will clear the current or existing data.
