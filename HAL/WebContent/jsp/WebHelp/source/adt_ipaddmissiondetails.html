IP Addmission Details

Screen Overview
This screen allows the admission of Patient. The admission clerk should check the current visit record whether the doctor has suggested for admission. HIN or service no. will be entered to enter the patient’s detail. The demographic details can also be capture through the Bar Coded OP label as an input. A unique admission number called the A&D No. will be given to the patient; It is a combination of Service type / Monthly Serial Number /Month / Year. for e.g. AF/1/12/2007, MT/2/12/2007 The system captures the Ward in which the patient is admitted. The bed is allotted for the patient from the list of unoccupied beds in the given ward on the basis of First in First out. The Patient status will be updated as “Admitted”.



Service Type :
The Service type of the Patient. It will be a Display field. It will be displayed on the basis of HIN.
Service Number :
It will be displayed on the basis of HIN. It is an unique number.
Service Status :
The service status of the service personnel. It will be a Display field. It will be displayed on the basis of HIN It cannot have null values

Relation :
The Relation to the service personnel. It will be a Display field. It will be displayed on the basis of HIN.
Rank :
The Rank of the service Person. It will be a Display field. It will be displayed on the basis of HIN.
Name of the service person :
The Name of the service person. It is a combination of Service Person First Name, Middle Name and Last Name. It will be a Display field. It will be displayed on the basis of HIN
Total Service :
The Total service in years and months of Service Personnel. It is enterable field. By default the data will fetch from Patient registration if the it is entered at the time of Registration. It Can have null Values.
Return FRW Issued :
FRW issued from parent unit. It is a selectable Field. It can not have null value.
FRW Sl.No. : If FRW Issued has been selected "Yes" then only it will be enable. Here FRW Sl. No. has to be entered. It is an Enterable field. It is mandatory in the case of FRW Issued = "Yes".
Place of Issue :
If FRW Issued has been selected "Yes" then only it will be enable. Here Place of FRW issue has to be entered. It is an Enterable field. It is mandatory in the case of FRW Issued = "Yes".
Unit :
The Unit of Service Person. This will be a list element (Selectable). The value will come from Unit Master. Along with selection there should be facility for manual entry if unit name is not available in the master. Master gets populated accordingly. By default the data will fetch from Patient registration if the it is entered at the time of Registration. It Can have null Values.
HIN No. :
It is an Enterable field. It is a system generated number And it is a combination of Service Type, Service status, Service number, Relation to the service personnel and Serial No. The Patient gets this number at the time of registration. IF the Patient has forgotten the HIN number then on the basis of Service number List of HIN will be displayed.
Sex : The Gender of the Patient. It will be a Display field. It will be displayed on the basis of HIN.

Patient Name : The name of the Patient. It is a combination of Patient’s First Name, Middle Name and Last Name. It will be a Display field. It will be displayed on the basis of HIN.
Age :
The age of the patient. It will be a Calculated field. The Age will be Calculated on the Basis of age entered at the time of registration and Current date. It will be a Display Field.
Religion :
The Religion of the patient. It will be a Display field. It will be displayed on the basis of HIN.
Patient Dist : The actual district at the time of Birth of the Patient. It will be a Display field. It will be displayed on the basis of HIN.
Adm Type : The type of Admission i.e. "General" , "Special" , New Born, etc. This will be a list element (Selectable) The value will come from Admission Type master. Default value will be "General". It cannot have null value.
Adm Date :
The date on which the Patient was admitted to the Hospital. By Default it will be Current system date It cannot have Null Values.
Adm Time :
The Time at which the Patient was Admitted to the Hospital. By Default it will be Current system time It cannot have Null Values.
Blood Group :
The Blood Group of the patient. This will be a list element (Selectable). The value will come from Blood Group master. It will be a Display and Updateable field. It will be displayed on the basis of HIN.
Ward :
The ward to which the patient is admitted in the hospital. This will be a list element (Selectable) The value will come from Department Master where department type is "Ward". It cannot have null
Admitted By :
The name of the Doctor. This will be a list element (Selectable) The value will come from employee master table where the employee type is "Doctor". It can have null Values.

Condition Status : Condition status will display as condition "Normal or Dead" is selected. And if condition "Critical" is selected then the combo will appear with value "SI list and DI List" where user can select the List. It can not have Null value.
Condition :
The Condition of the Patient at the time of admission This will be a list element (Selectable) The value will come from Condition master i.e. Normal, Critical, Dead, Walking, Laying. It Cannot have Null Values.
Case Type :
The Case type of the Patient. This will be a list element (Selectable) The value will come from Case Type Master. It can have null value.
Veg/Non-Veg :
The Diet of the Patient i.e. Vegetarian or Non-Vegetarian (V/NV) Default value will be "V" (Veg).
Patient Diet :
The Diet of the Patient given after Admission. This will be a list element (Selectable) The value will come from Diet Master. It can have null value.
Rcrd Off With Add :
The Record office with address of the Service Personnel. It will be a Display field. It will be displayed on the basis of HIN. Docmt. Init Document, which has been initiated at the time of Admission. It is enterable field. It Can have null value.
NOK Name :
The Patient Kin (Attendant Name ). It will be a Display and Updateable field. It will be displayed on the basis of HIN. It can have null values
NOK Relation :
The Relation of the patient with the Kin. This will be a list element (Selectable) The Value will come from Relation Master. It will be a Display and Updateable field. It will be displayed on the basis of HIN. It can have null Values.
NOK Add :
The Address of the Kin. It will be a Display and Updateable field. It will be displayed on the basis of HIN. It can have null Values.
NOK Phone :
The Phone no of the Kin. It will be a Display and Updateable field. It will be displayed on the basis of HIN. It can have null Values.
