Daily Diet & Extra Requisition

Screen Overview
This screen displays Daily diet and extras requisition. It also facilitates to change Lime & sugar for patients ,extras qty. & other details.



Ward Name:
User can select booing date details, it can’t remain blank.
MO i/c:
User can select the Medical office in charge details from the given list, this will come from employee screen and it can't remain blank.
Demand date:
It is system generated date and User can’t modify date, it can’t remain blank.
For date:
It is system generated date and User can’t modify date, it is one day next date of demand date.
Diets:
Based on ward name and demand date all diets details will display with veg and non-veg. It will calculate automatically.
Lime & Sugar for:
User can write required lime sugar details, it can remain blank.
Night Duty:
User can write night duty persons details, it can remain blank.
Other Diet:
User can write other diet details, it can remain blank.
Special Diet:
User can write any special diet details, it can remain blank.
Extras for N Diet:
Based on formula and based on N diet all patient extras for n diet item will calculate automatically, User can modify value and it can remain blank.
Supplementary Diets:
All supplementary diet details will display like officers and others. It will show present date and previous date and it will calculate automatically.
Therapeutic Diet:
All therapeutic diet details will display like officers and others. It will calculate automatically.

Bed state:

Transfer In:
Based on selection ward and date all transfer in patient count will display. It will calculate automatically based on transfer in from other wards.
Transfer Out:
Based on selection ward and date all transfer out patient count will display. It will calculate automatically based on transfer out from login ward.
Discharge:
Based on selection ward and date all discharges patient count will display. It will calculate automatically.
Death:
Based on selection ward and date all patient death count will display. It will calculate automatically.
Remaining:
Based on selection ward and date remaining patient list will display. It will calculate automatically.
Total Patient:
Based on selection ward and date total patient count will display. It will calculate automatically.
Save (Button):
By clicking on save button user can save the data.
Reset (Button):
Clicking on reset button, will clear the current or existing data.
