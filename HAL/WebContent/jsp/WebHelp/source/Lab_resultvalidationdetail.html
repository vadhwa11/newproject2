Result Validation Detail

Screen Overview
To capture the Details of pending results after the result entry.



Result Date:
It is result entry date, user can’t change date, it can't remain blank.
Result Time:
It is result entry time, user can’t change time, it can't remain blank.
Result Prepared By:
It is result prepared person name, user can’t change time, it can't remain blank.
Result Validated Date:
It is system date and today’s date, user can’t change date, it can't remain blank.
Result Validated Time:
It is system time and present time, user can’t change time, it can't remain blank.
Result Prepared By:
Default login person name will display, if user wants to change name user can change. It can't remain blank.
Service:
Based on order number test name will display
Sample:
Based on test sample will display from diagnosis master.
Result:
Result will come from result entry screen, if user wants user can change the result.
UOM:
Based on test UOM will display from diagnosis master.
Normal Range:
Based on test Normal Value will display from diagnosis master.
Validate:
User can validate single test or multiple tests.
Additional Remarks:
User can enter any observations.
Submit (Button):
By clicking on submit button user can save the data.
Reset (Button):
Clicking on reset button, will clear the current or existing data.
Back (Button):
Clicking on Back button, user will move to the previous screen.
