Indent To DGAFMS In The From Of SOC

Screen Overview
This screen captures the details of indent to DGAFMS in the form of SOC. SOC is generated by concerned department.



Indent No. :
Indent No. is generated automatically
Date of SOC :
Current date is displayed as SOC date.
Command :
By default CHAF, Bangalore is displayed by the system.
Hosp/Unit Address :
Hosp/ Unit address will be displayed by the system.
Item Dept. :
Select department name for which SOC is generated. It canâ€™t remain blank.
Details of patient :
Patient details should be entered.
Indent To :
By default system will display DGAFMS.

Item Details:

Sr No. :
By default sequence no from 1 will be displayed by the system.
PVMS No. :
It is code of indent item. Enter PVMS No. and press tab. Details of item will be displayed.
Nomenclature :
It is description of indent item. If PVMS No. is not entered then type nomenclature initial 2-3 characters, list will be displayed. After nomenclature selection A/U, Stock in and other details will be displayed.
A/U :
Accounting unit of PVMS No. Based on PVMS No. / nomenclature , A/U will be displayed.
Stock In :
Based on PVMS No. / nomenclature , stock will be displayed.
Unit Rate :
Based on PVMS No. / nomenclature , unit rate will be displayed.
Brand Name :
Based on PVMS No. / nomenclature , brand name will be displayed. Select brand name.
Manufactured by :
Based on PVMS No. / nomenclature & brand name, manufacture will be displayed.
PAC/Eqpt. Specific (For expendables) :
Select YES or No if item is PAC.
IF PAC, General detail justification :
User can write general detail justification.
Dose (for drugs) Day/Course :
User can write dose for no. of days.
Unit Rate :
Based on PVMS No. / nomenclature , unit rate will be displayed.
PVMS/NIV drugs substitute already prescribed :
User can write if drugs is already prescribed.
Justification of using NIV Item :
User can write justification if NIV item is prescribed.
Critical trial required :
User can select Yes/ NO if critical trial required.
Submit (Button) :
By clicking on submit button data will be saved.
Search :
By clicking on search , user can select previous saved data.
Print (Button) :
By clicking on print button report will be printed.
