Post OP Notes (Anaesthesia)

Screen Overview
This screen will capture patient post operative notes for Anaesthesia.



Yearly Serial No:
It is system generation number it will increase yearly, User can’t change number and it can't remain blank.
Monthly Serial No:
It is system generation number it will increase monthly, User can’t change number and it can't remain blank.
Date:
User can select OT Booking date, it can't remain blank.
Service Personnel Details:
On the basis of user selection patient service personnel details will appear and user can’t modify data.
Patient Details:
On the basis of user selection patient details will appear and user can’t modify data.
PAC Details (PAC Preview):
User can see PAC preview details.
Risk Grade:
User can select the risk grade from the given list, it can remain blank.
Surgery Name:
Based on surgery requisition surgery details will appear and user can add or delete the existing surgery details.
Surgeon’s Name:
Based on surgery requisition surgeon details will appear and user can add or delete the existing surgeon details.
Surgery from time:
User can write surgery starting time, it can remain blank.
Surgery to time:
User can write surgery closing time, it can remain blank.
Anaesthesia:
Based on PAC form patient anaesthesia details will appear and user can modify existing details.
Anesthesiologist(s):
Based on PAC form entry anesthesiologist details will appear and user can add or delete the existing details.
Anaesthesia from time:
User can write anaesthesia starting time, it can remain blank.
Anaesthesia to time:
User can write anaesthesia closing time, it can remain blank.
Pre medication/Induction:
Based on PAC form entry anesthesiologist details will appear and user can add or delete the existing details.
General Examination:
User can write patient general examination details like Pulse, Icetrus, Nourishment, Pallor, Oedema, Bp, Cyanosis, Spine, Airway, Clubbing, Thyroid, Venous Access. And all can remain blank
Type:
User can select from the give list, it contains pre medicine or induction and it can remain blank.
Nomenclature:
User can select nomenclature from the give list, it can remain blank. User can add more medicines in the list.
PVMS/NIV No:
Based on nomenclature pvms / niv number will come automatically, user can’t change value and it can’t remain blank.
Dose:
User can write dose of medicine, it can remain blank.
Route:
User can select from the give list, it can remain blank.
ETT/LMA:
User can select from the give list, it contains ETT size, LMA size. It can remain blank.
Maintenance:
User can select maintenance like Halothane, Isoflurane, Sevoflurane, NitronsOxide, Profofol, Ketamine, Bupivacaine 0.5%

Procedure Details:

Anesthesia:
User can select combination of anesthesia details.
Nomenclature:
User can select nomenclature from the give list, it can remain blank. User can add more medicines in the list.
PVMS/NIV No:
Based on nomenclature pvms / niv number will come automatically, user can’t change value and it can’t remain blank.
Dose:
User can write dose of medicine, it can remain blank
Level:
User can select from the give list, it can remain blank.
Catheter:
User can write catheter details, it can remain blank.
Monitoring:
User can select monitoring like ECG, NIBP, CVP, Temp, SpO2, IABP, UO.

IV Fluids:

Nomenclature:
User can select nomenclature from the give list, it can remain blank. User can add more medicines in the list.
PVMS/NIV No:
Based on nomenclature pvms / niv number will come automatically, user can’t change value and it can’t remain blank.
Fluid Name:
User can write fluid name, it can remain blank
Volume (ml):
User can write volume usage for particular drug, it can remain blank.
Neostigmine:
User can write neostigmine details it can remain blank.
Glycophyrrolate:
User can write glycophrrolate details it can remain blank.
Others:
User can write any other details it can remain blank.
Recovery:
User can write recovery details it can remain blank.
E/ Others:
User can select from the given list, It contains E or Others and it can remain blank.
Remarks:
User can write any remarks or any observations, it can remain blank.
Submit (Button):
By clicking on Submit button user can save the data.
Back (Button):
Clicking on Back button, will move user to the previous screen.
