Result Entry Detail

Screen Overview
To capture the Details of result entered after sample validation



Result Date:
It is system date and today’s date, user can’t change date, it can't remain blank.
Result Time:
It is system time and present time, user can’t change time, it can't remain blank.
Result Prepared By:
Default login person name will display, if user wants to change name user can change. it can't remain blank.
Sample Validation Date:
It is sample validated date, user can’t change date, it can't remain blank.
Sample Validation Time:
It is sample validated time, user can’t change time, it can't remain blank.
Sample Validated By:
It is sample validated person name, it can't remain blank.
Diag No:
It is diagnosis number, it will come from sample validated screen. It can't remain blank.
Service:
Based on order number test name will display, it can't remain blank.
Sample:
Based on test sample will display from diagnosis master.
Result:
Diagnosis test is based on result type, if result type is template then template screen will appear. If result type is Sensitive then Culture and sensitive screen will appear.
UOM:
Based on test UOM will display from diagnosis master.
Normal Range:
Based on test Normal Value will display from diagnosis master.
Additional Remarks:
User can write any observations.
Patient History:
User can view patient previous history diagnosis details.
Submit (Button):
By clicking on save button user can save the data.
Reset (Button):
Clicking on reset button, will clear the current or existing data.
Back (Button):
Clicking on Back button, user will move to the previous screen.en.
